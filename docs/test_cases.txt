3. CASOS DE PRUEBA DE SISTEMA (SYSTEM / E2E TESTS)
---------------------------------------------------------------------
Propósito: Validar flujos de trabajo completos de extremo a extremo.

CP-SIS-01: Ciclo de Vida de Alerta de Crisis (Flujo Crítico)
- Pasos: 
  1. Usuario escribe "Me siento muy mal, ya no quiero seguir".
  2. IA detecta 'Crisis'.
  3. Admin recibe alerta en el Panel.
  4. Admin cambia estado a "En Proceso".
- Resultado Esperado: El flujo se completa sin errores y el estado persiste para el Admin.

CP-SIS-02: Consistencia Visual Multi-Modo (UI/UX)
- Pasos:
  1. Abrir Estadísticas en Modo Claro.
  2. Cambiar a Modo Oscuro desde Perfil.
  3. Regresar a Estadísticas.
- Resultado Esperado: Las gráficas y tarjetas deben adaptar sus colores inmediatamente sin recargar la app.

CP-SIS-03: Seguridad de Roles (RBAC)
- Pasos:
  1. Login con cuenta de rol 'user'.
  2. Intentar navegar manualmente a la ruta de Admin.
- Resultado Esperado: El sistema debe redirigir al Home o denegar el acceso mediante las políticas RLS de Supabase.

CP-SIS-04: Robustez en Modo Offline
- Pasos:
  1. Desactivar internet.
  2. Escribir entrada de diario.
  3. Cerrar y abrir la app.
- Resultado Esperado: El borrador (Draft) debe estar disponible gracias al guardado en AsyncStorage.

CP-SIS-05: Integridad de Reportes Exportables
- Pasos:
  1. Admin hace clic en "Exportar Reporte Mensual".
  2. Seleccionar compartir vía Email.
- Resultado Esperado: Se genera un cuerpo de texto coherente que incluye el conteo de usuarios y la lista de alertas activas.
